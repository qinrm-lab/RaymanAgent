=======================================================
🤖 Rayman AI 助手 - 快捷指令列表
=======================================================

[环境初始化]
▶ 初始化 / 运行 setup
  - 作用：自动检测项目名，并生成/更新 `.SolutionName`, `.github/copilot-instructions.md` 和 `.vscode/tasks.json`。
  - 默认：初始化结束后会自动执行 `Release Gate`（project 模式）；可通过 `-SkipReleaseGate` 跳过。
  - Git bootstrap：若当前工作区没有 `.git`，setup 默认会自动初始化本地 Git 仓库，并尽量把初始分支固定为 `main`。
  - GitHub 登录：交互式 setup 默认会尝试执行 `gh auth login --web` 与 `auth setup-git`；失败默认只告警，不阻断本地 Git，除非显式开启严格模式。
  - 不会自动创建 remote，也不会自动设置 `origin`。
  - 产物：Git / GitHub 引导报告会写入 `.Rayman/runtime/git.bootstrap.last.json`。
  - 向量库：默认使用根目录 `.rag/<命名空间>/chroma_db`；若检测到已有库则复用并跳过重建，仅在 `-ForceReindex` 时重建。
  - 迁移策略：默认会自动迁移旧路径 `.Rayman/state/chroma_db` 到新路径；如需关闭可用 `-NoAutoMigrateLegacyRag`。
  - SCM 噪声治理：setup 末尾会自动注入忽略规则（external 默认 `.gitignore`，source 默认 `.git/info/exclude`），并输出 `modified/added/untracked/total` 统计与软阈值告警（默认 100）。
  - 相关变量：`RAYMAN_SETUP_GIT_INIT=0|1`、`RAYMAN_SETUP_GITHUB_LOGIN=0|1`、`RAYMAN_SETUP_GITHUB_LOGIN_STRICT=0|1`、`RAYMAN_GITHUB_HOST=<host>`、`RAYMAN_GITHUB_GIT_PROTOCOL=https`、`RAYMAN_SCM_IGNORE_MODE=info-exclude|gitignore|off`、`RAYMAN_SCM_CHANGE_SOFT_LIMIT=100`、`RAYMAN_ALLOW_TRACKED_RAYMAN_ASSETS=0|1`。
  - 注意：若 `.Rayman/**`、`.SolutionName`、`.cursorrules`、`.clinerules`、`.rayman.env.ps1`、`.github/copilot-instructions.md` 已被 Git 跟踪，external workspace 会严格阻断；source workspace 允许 authored `.Rayman/` 与 `.SolutionName`，但仍会阻断本地/生成资产。
  - 说明：`.dotnet10/`、`dist/`、`publish/`、`.tmp/`、`.temp/` 等常见产物目录若被 Git 跟踪，仅告警不阻断。
  - 场景：当你将 `.Rayman` 文件夹拷贝到一个新项目时，首次运行以激活所有功能。

▶ WSL 安装依赖 / 安装 WSL 依赖
  - 作用：在 Ubuntu(WSL2) 中自动安装 `pwsh`、`notify-send`、语音工具等（需要 sudo）。
  - 场景：你在 WSL 下打开 `/mnt/*` 工作区，但任务提示找不到 `pwsh` / 通知/语音不可用。
  - 相关任务：`Rayman: Ensure WSL Deps`

▶ Windows 依赖检查
  - 作用：检查 Windows 侧常用工具（git/wsl/winget 等）是否存在，并给出提示。
  - 手动运行：`.Rayman/rayman.ps1 ensure-win-deps`

[任务状态管理]
▶ 保存状态 / 暂停
  - 作用：保存任务状态到 `.Rayman/state/pending_task.md`，并自动 `git stash`（硬状态保存代码变更）。
  - 场景：当你需要离开，或者想中断当前长任务时使用（请先点击停止生成按钮）。

▶ 继续
  - 作用：读取 `.Rayman/state/pending_task.md` 并恢复任务；同时自动 `git stash pop` 恢复代码变更。
  - 场景：当你重新打开 VS Code，看到有未完成任务的提醒时使用。

▶ 清除保存的状态
  - 作用：彻底删除之前保存的任务状态，取消启动时的未完成任务提醒。
  - 场景：当你决定放弃之前中断的任务时使用。

[后台服务管理]
▶ 停止监听 / 停止后台服务
  - 作用：停止所有在后台运行的 Rayman 监听脚本（如 prompt-watch, attention-watch），并回收当前 workspace/当前 VS Code owner 下由 Rayman 登记的 `.NET` 业务残留进程。
  - 场景：当你需要释放系统资源，或者想手动重启监听服务时使用。

[项目运维]
▶ 自愈 / 测试并修复
  - 作用：自动检测项目类型并运行构建/测试；失败时生成 `.Rayman/state/last_error.log` 和 `.Rayman/state/last_error_summary.md` 供 AI 读取后尝试修复。
  - 默认策略：自动补全缺失依赖（dotnet/node/python/playwright）；MAUI 项目会额外执行 `dotnet workload restore`，仍缺失时阻断并给出日志入口。
  - 可选关闭：手动设置 `RAYMAN_AUTO_INSTALL_TEST_DEPS=0`（或对应变量）后可临时关闭自动补全。
  - .NET（WSL on Windows）：默认优先在 Windows Host 执行；失败时按 `RAYMAN_DOTNET_WINDOWS_STRICT` 决定是否回退 WSL。
  - MAUI：默认按宿主选择单一 target framework 构建；Windows/Windows bridge 优先 `net*-windows`，macOS 优先 `net*-maccatalyst`，其余宿主优先 `net*-android`。
  - 排障入口：`.Rayman/runtime/dotnet.exec.last.json` 与 `.Rayman/logs/dotnet.exec.<timestamp>.log`；前者会在需要时补充 `owned_root_pid`、`cleanup_attempted`、`cleanup_result`、`cleanup_reason`、`cleanup_pids`。
  - 场景：当你怀疑依赖/构建/测试失败，想让 AI 进入“测试-修复-再测”的闭环时使用。

▶ Agent 路由 / dispatch
  - 作用：按任务类型路由到 `copilot|codex|local`，并自动处理云端白名单与回退。
  - 系统精简：当前仅精简 `dispatch`。满足版本门槛时会自动委派 `codex exec`（保留原任务入口与标签）。
  - Capability 提示：OpenAI/API/docs 类任务会优先提示 OpenAI Docs MCP；browser/web/e2e 类任务会优先提示 Playwright MCP；WinForms / MAUI(Windows) / desktop / UIA 类任务会优先提示 Rayman WinApp MCP，并在 `.Rayman/runtime/agent_runs/*.json` 记录命中结果。
  - 审计报告：`.Rayman/runtime/system_slim.report.md`（含“已精简功能/未精简原因/版本变化提醒”）。
  - 产物：`.Rayman/runtime/agent_runs/<run_id>.json` 与 `last.json`。
  - 手动运行：`.Rayman/rayman.ps1 dispatch --TaskKind bugfix`
  - 紧急绕过（审计）：`.Rayman/rayman.ps1 dispatch --TaskKind bugfix -PolicyBypass -BypassReason "紧急发布修复"`（会写入 `.Rayman/runtime/decision.log`）

▶ Agent Capabilities
  - 作用：同步/查看 Rayman 声明式 capability registry，并维护工作区级 `.codex/config.toml`。
  - 默认能力：`openai_docs`（OpenAI Docs MCP）、`web_auto_test`（Playwright MCP + Rayman fallback）、`winapp_auto_test`（Rayman WinApp MCP + Rayman fallback）。
  - 产物：`.Rayman/runtime/agent_capabilities.report.json`、`.Rayman/runtime/agent_capabilities.report.md`、`.codex/config.toml`。
  - 手动运行：
    - `.Rayman/rayman.ps1 agent-capabilities`
    - `.Rayman/rayman.ps1 agent-capabilities --sync`

▶ Windows 桌面自动化 / WinApp
  - 作用：使用 Windows UI Automation 自动化 WinForms / MAUI(Windows) 桌面应用，不依赖 Appium / WinAppDriver。
  - 就绪检查：`.Rayman/rayman.ps1 ensure-winapp`
  - 运行 flow：`.Rayman/rayman.ps1 winapp-test --flow-file .Rayman/winapp.flow.sample.json`
  - 控件树探查：`.Rayman/rayman.ps1 winapp-inspect --window-title-regex ".*"`
  - 产物：`.Rayman/runtime/winapp.ready.windows.json`、`.Rayman/runtime/winapp-tests/last_result.json`、`.Rayman/runtime/winapp-tests/control_tree.json`

▶ 评审闭环 / review-loop
  - 作用：执行“dispatch → test-fix → 再测”迭代，最多 `RAYMAN_REVIEW_LOOP_MAX_ROUNDS` 轮（保持 Rayman 闭环，不替换为 codex review）。
  - 产物：`.Rayman/runtime/review_loop.last.json`、`.Rayman/runtime/review_loop.last.diff.md`、`.Rayman/runtime/telemetry/first_pass_runs.tsv`。
  - 手动运行：`.Rayman/rayman.ps1 review-loop --TaskKind review`

▶ 一次通过率报告 / first-pass-report
  - 作用：输出一次通过率 KPI（窗口默认 `RAYMAN_FIRST_PASS_WINDOW=20`）。
  - 产物：`.Rayman/runtime/telemetry/first_pass_report.md`、`.json`（含 round1 改动规模相关性分析）。
  - 手动运行：`.Rayman/rayman.ps1 first-pass-report`

▶ Prompt 模板 / prompts
  - 作用：列出、查看或应用 `.github/prompts/*.prompt.md` 任务模板。
  - 手动运行：
    - `.Rayman/rayman.ps1 prompts -Action list`
    - `.Rayman/rayman.ps1 prompts -Action apply -Name bugfix.prompt.md -Task "..." -AcceptanceCriteria "..."`

▶ 更新上下文
  - 作用：生成/更新 `.Rayman/CONTEXT.md`（项目结构快照），供 AI 在复杂任务前快速对齐项目现状。
  - 场景：项目结构变化较大、AI 需要更准确上下文时使用。

▶ 清理缓存 / 一键清理缓存
  - 作用：清理项目中的临时文件和缓存（如 bin, obj, node_modules, .vs 等）。
  - 场景：当项目编译报错、缓存冲突或需要释放磁盘空间时使用。

▶ 一键部署 [项目名]
  - 作用：执行项目的构建、打包和部署流程。
  - 场景：当代码开发完成，需要发布到测试或生产环境时使用。
  - 示例：输入“一键部署”会部署所有项目；输入“一键部署 WebApp”只部署 WebApp 项目。

▶ 发布闸门 / Release Gate
  - 作用：执行发布前强校验，生成 `.Rayman/state/release_gate_report.md`（PASS/WARN/FAIL）。
  - 模式：支持 `-Mode project`（推荐用于业务项目）和 `-Mode standard`（仓库治理模式）。
  - 场景：准备将 `.Rayman` 拷贝到真实环境前，先做一键发布就绪验收。
  - 手动运行：`.Rayman/rayman.ps1 release-gate`

▶ 代理健康检查 / proxy-health
  - 作用：同时展示用户设置、工作区设置、系统代理、WinHTTP 与当前进程环境里的代理候选值，并显示最终生效 snapshot。
  - 场景：代理类型/端口经常变化时，快速定位“写在哪里”和“最终读到了什么”。
  - 手动运行：`.Rayman/rayman.ps1 proxy-health --refresh`

▶ 拷贝初始化烟测 / doctor --copy-smoke
  - 作用：复制 `.Rayman` 到临时目录并执行一次 setup，快速验证“拷贝后是否可初始化”。
  - 默认：会净化 `.Rayman/runtime` 与 `.Rayman/logs` 历史产物后再执行（减少旧环境噪声）。
  - 可选：`--strict` 使用严格路径（不跳过 release gate / 不放宽环境）。
  - 可选：`--scope wsl|host|all|sandbox` 指定 Playwright 检查范围（默认 `wsl`，WSL 不可用时可显式用 `host`）。
  - 手动运行：`.Rayman/rayman.ps1 doctor --copy-smoke`

▶ 拷贝后初始化自检 / copy-self-check
  - 作用：`doctor --copy-smoke` 的快捷别名，一条命令完成“拷贝后初始化可用性”验收。
  - 严格模式：除首次 setup 成功外，还会验证 external workspace 命中 tracked Rayman assets 时会先阻断、显式设置 `RAYMAN_ALLOW_TRACKED_RAYMAN_ASSETS=1` 后才放行；同时验证 source workspace 允许 authored `.Rayman/` / `.SolutionName` 但仍阻断本地/生成资产。
  - 可选参数：`--strict`、`--scope wsl|host|all|sandbox`、`--timeout-seconds <秒>`、`--keep-temp`。
  - 手动运行：`.Rayman/rayman.ps1 copy-self-check`

▶ 项目快速门禁 / fast-gate
  - 作用：统一的 PR 级快速门禁入口，输出 `.Rayman/runtime/project_gates/fast.report.json`。
  - 默认检查：requirements layout、protected assets、`build_command`、`extensions.project_fast_checks`。
  - 项目扩展：通过工作区根目录 `.rayman.project.json` 配置，不建议直接改 workflow。
  - 手动运行：`.Rayman/rayman.ps1 fast-gate`

▶ 项目浏览器门禁 / browser-gate
  - 作用：统一的浏览器 / 项目冒烟门禁入口，输出 `.Rayman/runtime/project_gates/browser.report.json`。
  - 默认基线：Chromium；项目可在 `.rayman.project.json` 的 `browser_command` 中覆盖具体实现。
  - 手动运行：`.Rayman/rayman.ps1 browser-gate`

▶ 项目完整门禁 / full-gate
  - 作用：统一的 nightly / manual / 发布前门禁入口，输出 `.Rayman/runtime/project_gates/full.report.json`，并联动 `release-gate -Mode project`。
  - 扩展点：`full_gate_command` 与 `extensions.project_release_checks`。
  - 手动运行：`.Rayman/rayman.ps1 full-gate`

▶ 严格出厂验收 / copy-self-check --strict
  - 作用：严格模式自检（包含 Release Gate），用于发给同事前的一键出厂验收。
  - 默认任务：`Rayman: Factory Check (Strict)`（默认附带 `--scope wsl`，减少无 Sandbox 机器误阻断）。
  - 手动运行：`.Rayman/rayman.ps1 copy-self-check --strict --scope wsl`

▶ 严格出厂验收（保留现场）
  - 作用：严格模式下保留临时验收目录；失败时自动打开目录，便于同事立刻排障。
  - 失败增强：会自动把关键路径（临时目录/setup 日志/Playwright 摘要/Release Gate 报告）复制到剪贴板。
  - 默认任务：`Rayman: Factory Check (Strict, Keep Temp)`。
  - 手动运行：`.Rayman/rayman.ps1 copy-self-check --strict --scope wsl --keep-temp --open-on-fail`

▶ Requirements 时间戳回填 / req-ts-backfill
  - 作用：为 `.<SolutionName>/` 下全部 `*.requirements.md` 回填缺失时间戳（块元数据 + 可视内容行），并更新同步元数据。
  - 场景：从旧版规则升级到新版“按时间同步”策略时，一键补齐历史文件。
  - 手动运行：`.Rayman/rayman.ps1 req-ts-backfill`

▶ 迁移旧向量库 / migrate-rag
  - 作用：把旧路径 `.Rayman/state/chroma_db`（及 `rag.db`）迁移到 `.rag/<命名空间>/`。
  - 场景：从旧版 Rayman 升级到新路径结构时，一键迁移历史向量库。
  - 手动运行：`.Rayman/rayman.ps1 migrate-rag`

▶ 生成可分发包 / Package Dist
  - 作用：生成精简版 `.Rayman` 分发 zip，默认保留核心能力并剔除向量库/运行时/日志等临时数据。
  - 净化：即便指定 `-IncludeRuntimeFiles/-IncludeLogs`，默认仍净化历史 runtime/logs 产物；如需保留历史可加 `-KeepRuntimeHistory/-KeepLogHistory`。
  - 场景：准备拷贝到新项目或发给团队成员时，避免携带大体积临时文件。
  - 手动运行：`.Rayman/rayman.ps1 package-dist`

▶ 每日健康检查（自动）
  - 作用：按 freshness 运行健康检查；默认 24h 内只跑一次，或关键输入变化时重跑。
  - 扫描范围：默认排除 `.Rayman/.dist`、`.Rayman/runtime`、`.Rayman/state`、`.Rayman/logs`、`node_modules`、`.artifacts`、`bin`、`obj`、`test-results` 等生成目录。
  - 默认：会自动执行 requirements 时间戳回填（`req-ts-backfill` 逻辑）。
  - 开关：设置环境变量 `RAYMAN_DAILY_REQ_TS_BACKFILL_ENABLED=0` 可关闭。
  - 可选守护：设置 `RAYMAN_DAILY_WINAPP_GUARD_SMOKE_ENABLED=1` 可在健康检查中附加执行 WinApp Guard 串行烟测（正例应通过、负例应失败）。
  - 手动运行：`.Rayman/rayman.ps1 health-check`

▶ VS Code folderOpen bootstrap
  - 作用：默认 profile=`conservative`，同步必跑 watcher/session 去重与 pending task；`daily health`、`context refresh`、轻量依赖检查改为 stale-only。
  - runtime 报告：`.Rayman/runtime/vscode_bootstrap.last.json`
  - 可选变量：`RAYMAN_VSCODE_BOOTSTRAP_PROFILE=conservative|active|strict`

[帮助]
▶ ? 或 ？

▶ 运行 `.Rayman/rayman.ps1`（无参数）
  - 作用：进入交互式编号菜单，直接选命令执行，减少参数输入错误；会记住上次选择并在下次回车默认执行。
  - 场景：不想记参数时，推荐用这个入口。
